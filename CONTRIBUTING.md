# Contributing Guide

Thank you for your interest in contributing.

We welcome contributions that improve the project, fix bugs, enhance documentation, or add valuable new features.

---

## Before Contributing

Please:

* Read the README carefully
* Search existing issues before opening a new one
* Open an issue first for major changes
* Keep pull requests focused and small when possible

---

## Development Setup

Clone the repository:

```bash
git clone <repository-url>
cd project-name
```

Install dependencies:

```bash
npm install
```

Run tests:

```bash
npm test
```

---

## Branch Naming

Recommended format:

```text
feature/add-refresh-rotation
fix/token-validation-bug
docs/update-readme
```

---

## Commit Messages

Use clear commit messages.

Recommended style:

```text
feat: add refresh token rotation
fix: resolve token expiration issue
docs: improve installation guide
refactor: simplify auth middleware
```

---

## Pull Requests

Before submitting a PR:

* Ensure tests pass
* Update documentation if needed
* Keep PR focused on one concern
* Explain clearly what changed and why

---

## Reporting Bugs

Please include:

* Expected behavior
* Actual behavior
* Steps to reproduce
* Environment details
* Node.js version
* Package version

---

## Feature Requests

Feature requests are welcome.

Please explain:

* The problem you are solving
* Why the feature is useful
* Possible implementation ideas

---

## Questions

If you are unsure about something, open an issue first.

We prefer discussion before large implementations.

Thank you for contributing.
